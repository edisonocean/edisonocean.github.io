What is the Enumerable "group_by"

Mon, January 19th, 2015

Let's first start with explaining what is "Enumerable" is. Well, before it gets confusing, I'll put it in simple terms. "Enumerable" is a collections of methods that can interact, meaning "included with", with other classes. By implementing them in the beginning of our code it allows us to only have to use them once instead of having to define them each time we want to use them again. That is in short what programmers refer to "inheritance". Since coding is like a family tree, the rich grand, grand, grand, grand, grandad code can pass down it's riches to the younger code.

"Enumerable" has in it's toolbox the method #each. It can be used with arrays, hashes, ranges, etc. It has the ability to go through each item in, let's say, an array and cast whatever condition was previously defined or following the method #each. By "definded" I mean, whatever action you want each item in the array (aka list) to do. Now let's move on to the method "group_by".

For example:

  1. method = [1, 2, 3, 4, 4, 4, 5, 5, 6, 7, 8]
  2. method.each do |i| #Here we're saying: 'for every item in the array called 'method' use 'i' to represent it from now on.'
  3. i = i + 2 #Because we're using '.each' here we're adding 2 to every number in the array.
  4. return i #This will return the new value of each item on the list
  5. end

The method "group_by" will run the condition or conditions that that come after it in the code you wrote. However, after running that code, it will return a "hash". So in the example above we can use "sort_by" like this:

  1. method = [1, 2, 3, 4, 4, 4, 5, 5, 6, 7, 8]
  2. method.group_by{|number| number > 3}
  3. end
This code will return an array of the numbers greater than '3' from the array called 'method'. Like this: {4, 4, 4, 5, 5, 6, 7, 8}
