TECH BLOG: arrays AND hashes!!

Friday, January 9th, 2015

Hello Folks!!!, Today we'll be reviewing Arrays and Hashes... I know, you're all besides yourselves. Ok, I know you'll make a ton of friends but, settle down. Now, in this Ruby world that we are in, when we say the word "arrays" what we're talking about is a sequence of numbers, letters, words, etc. or a sequence of sequences of numbers,letters, words, etc. But it gets more involved than that. Another way to look at it is that it's a manner in arranging all these things I just mentioned. Something that's related to this is "hashes". Creating a hash allows us to do the same thing but in a slightly different way. All in all, we can say that by using one or the other, we can achieve the same result. I'll try to keep it as simple as possible in this blog. However, if you feel like you need further explanation, I'll also provide links to resources that I feel go further into describing what these two super heroes are capable of doing.

What are arrays?

Saying the word 'arrays' is just another way of saying ['a word', 'or written number like 'four'', 'or an integer like '3'', ['or a sequence in brackets inside', 'a sequence, that's already', 'in brackets', 'like this one'] ]. As you can see, there are square brackets that I used to contain a list of things. Inside those brackets, they were separated by commas, and each series of words that I called "things" is called a string. A string is each collection of words that I wrapped in single quotes like this 'blah blah blah'. An example of what an array might actually look like in a document goes like this:

array_1 = ['hi', 45, 'my favorite', 'number', ['this', 'is', 2], ['arrays', 'inside', 1, 'array']] .

For us to use this array in a funtional way, we need to say:

puts array_1 [0]
puts array_1 [6]
puts array_1 [3]
puts array_1 [2]
puts array_1 [6]

This will return (print out) the items inside the array called 'array_1' in the order as specified by the numbers inside the square brackets '[#]'. Notice that there is a zero inside one of the square brackets? In the programming world, the first item in a list, starts with zero. So, in array_1, the first item is 'hi' and its place in the lineup is zero.

Now... a hash is a different way of doing the same thing. Some would say, it's easier; Yet, others would say it's not any easier; However, in the end, the product is just the same. I see it as just two different flavors and either way, you'll get to eat. However, as humans are said to have 5 senses, hashes seem to have at least one more sense than arrays do. Here is an example of a hash.

hash = {
key1 => value1,
key2 => value2,
key3 => value3
}

puts hash["key1"]
puts hash["key2"]
puts hash["key3"]

Let's see if I can explain this. 'hash' is, at this point, a variable that is representing all the 'key#'s inside the curly brackets. It's like their lawyer :-). The 'key#'s are then pointing at their respective values. Now, what makes this easy to use, is the fact that we can call on 'hash' to print out the keys. We just need to memorize the syntax on how to do it. So:

puts hash["key1"] will return (print out) 'value1'

It's as simple as that.
